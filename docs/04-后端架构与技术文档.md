## 二、模块职责划分

### 2.1 配置层 `config/`

| 文件 | 职责 |
|------|------|
| `settings/__init__.py` | 加载 `.env` 到环境变量，根据 `DJANGO_ENV` 选择开发/生产配置 |
| `settings/base.py` | 定义 `INSTALLED_APPS`、中间件、数据库连接、`DATABASE_CONFIG`、`LLM_CONFIG`、`MCP_CONFIG`、`INVITATION_CODE` |
| `settings/development.py` | `DEBUG=True`、CORS全开放、MySQL 配置 |
| `urls.py` | 全局路由入口，将 `/api/v1/` 分发到各 App 的 urls |

### 2.2 用户模块 `apps/face/`

**职责**: 用户注册、登录（密码/人脸）、个人中心管理

| 文件 | 职责 |
|------|------|
| `models.py` | `UserInfo` 模型（user_id, user_name, user_age, user_phone, user_pwd） |
| `views.py` | `RegisterView`、`PasswordLoginView`、`FaceLoginView`、`LogoutView`、`get_login_user_id()` |
| `views_profile.py` | `UserProfileView`（查询/更新用户信息） |
| `services.py` | `FaceService`（人脸识别）、`DatabaseService`（MySQL CRUD）、`RandomService`（唯一ID） |
| `forms.py` | `UserRegisterForm`、`UserLoginForm` |
| `apps.py` | `FaceConfig`——Django 启动时自动启动 MCP Server 子进程 |

### 2.3 聊天模块 `apps/chat/`

**职责**: 多会话管理、消息CRUD、Agent SSE 流式对话（核心业务流程）

| 文件 | 职责 |
|------|------|
| `models.py` | `ChatSession`（会话）、`ChatMessage`（消息，含 thinking_process, echarts_config） |
| `views.py` | `SessionListView`（会话CRUD）、`SessionMessagesView`（消息列表）、`SessionDeleteView`（删除会话）、`ChatAgentView`（Agent SSE 流式对话） |

### 2.4 MCP工具模块 `apps/mcp/`

**职责**: 直接调用 MCP-Jobs 服务，提供职位搜索和投递 API（绕开 Agent，直接调用工具）

| 文件 | 职责 |
|------|------|
| `models.py` | `MCPJobsCache`（职位数据缓存表，7天过期） |
| `views.py` | `JobsView`（搜索职位+缓存）、`ApplyJobView`（投递职位） |

### 2.5 Agent 层 `ai/`

**职责**: AI 智能体核心——LLM 加载、工具注册、提示词管理

| 文件 | 职责 |
|------|------|
| `prompts/system_prompt.py` | 职途AI 系统提示词（角色定位、工具列表、工作流程、输出格式、核心原则） |
| `agents/loader.py` | `LLMService` 单例（通义千问 qwen3-max）、`get_database` 工具、`load_agent()` |
| `mcp/client.py` | MCP 客户端（连接 MCP-Jobs 服务，获取招聘搜索工具） |
| `mcp/server.py` | MCP 服务端（FastMCP，提供 `query_database` 本地工具） |

### 2.6 公共层 `core/`

**职责**: 通用工具函数、自定义异常、表单验证

## 四、数据库表设计

### 4.1 user_info（用户信息表）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | INT (PK) | 用户ID（随机生成，1-100） |
| user_name | VARCHAR(100) UNIQUE | 用户名 |
| user_age | INT DEFAULT 0 | 年龄 |
| user_phone | VARCHAR(20) INDEX | 手机号 |
| user_pwd | VARCHAR(100) | 密码（明文存储，后续优化为bcrypt） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 4.2 chat_session（聊天会话表）

| 字段 | 类型 | 说明 |
|------|------|------|
| session_id | BIGINT (PK) | 会话ID（自增） |
| user_id | INT (FK→user_info) | 所属用户（级联删除） |
| title | VARCHAR(200) | 会话标题 |
| last_message_time | DATETIME | 最后消息时间（自动更新） |
| created_at | DATETIME | 创建时间 |

索引: `(user_id, last_message_time)`

### 4.3 chat_message（聊天消息表）

| 字段 | 类型 | 说明 |
|------|------|------|
| message_id | BIGINT (PK) | 消息ID（自增） |
| session_id | BIGINT (FK→chat_session) | 所属会话（级联删除） |
| user_id | INT (FK→user_info) | 所属用户（级联删除） |
| role | VARCHAR(20) | `user` 或 `assistant` |
| content | TEXT | 消息内容 |
| thinking_process | TEXT NULL | 思考过程（深度思考开启时存储，关闭时NULL） |
| echarts_config | JSON NULL | ECharts 图表配置 |
| created_at | DATETIME | 创建时间 |

索引: `(session_id, created_at)`

### 4.4 mcp_jobs_cache（职位数据缓存表）

| 字段 | 类型 | 说明 |
|------|------|------|
| cache_id | BIGINT (PK) | 缓存ID（自增） |
| keyword | VARCHAR(200) | 岗位关键词 |
| city | VARCHAR(100) | 城市 |
| recruit_type | INT NULL | 职位类型（0-社招, 1-校招, 2-实习） |
| jobs_data | JSON | 职位数据 |
| expire_at | DATETIME | 过期时间（创建时间+7天） |
| created_at | DATETIME | 创建时间 |

唯一约束: `(keyword, city, recruit_type)`  
索引: `(expire_at)`

## 五、完整数据流（核心业务流程）

### 5.1 用户注册流程

```
前端 → POST /api/v1/auth/register/
       Content-Type: application/json
       Body: {
         "user_name": "张三",
         "user_pwd": "abc123",
         "user_age": 22,
         "user_phone": "13800138000",
         "invitation_code": "ABC123",
         "face_image": "data:image/jpeg;base64,..."  // 可选
       }

       ← 200 OK
       Body: {
         "code": 200,
         "msg": "success",
         "data": { "user_id": 42 }
       }

处理流程:
1. RegisterView.post() 接收请求
2. 验证邀请码 (settings.INVITATION_CODE)
3. UserRegisterForm 验证用户名和密码格式
4. 检查用户名是否已存在
5. 如果包含 face_image:
   - Base64 解码图像
   - FaceService.get_face_encoding() 提取人脸特征
   - 遍历已有人脸图片，检查是否重复采集
   - 生成唯一ID (1-100)，保存人脸图片到 MEDIA_ROOT/face_images/{id}.jpg
6. DatabaseService.insert_user() 写入 MySQL
7. 返回 user_id
```

**错误码**:
| code | 说明 |
|------|------|
| 1001 | 邀请码无效 |
| 1002 | 用户名已存在 |
| 1003 | 密码格式错误 |
| 400  | 未检测到人脸 / 已采集过人脸 |
| 500  | 服务器内部错误 |

### 5.2 用户登录流程

**密码登录**:
```
前端 → POST /api/v1/auth/login/password/
       Body: {
         "login_account": "张三或13800138000",
         "user_pwd": "abc123"
       }

       ← 200 OK
       Body: {
         "code": 200,
         "msg": "success",
         "data": {
           "user_id": 42,
           "user_name": "张三",
           "user_age": 22,
           "user_phone": "13800138000"
         }
       }

处理流程:
1. PasswordLoginView.post() 接收请求
2. 先按 user_name 查询，未命中则按 user_phone 查询
3. 验证密码（明文比对）
4. 设置 session: request.session['user_id'] / request.session['user_name']
5. 返回用户信息（不含密码）
```

**人脸登录**:
```
前端 → POST /api/v1/auth/login/face/
       Body: {
         "face_image": "data:image/jpeg;base64,..."
       }

       ← 200 OK (同上) 或 1005 未识别到已注册人脸

处理流程:
1. FaceLoginView.post() 接收请求
2. Base64 解码 → 提取人脸特征
3. 遍历 face_images/ 目录下所有图片，逐一比对
4. 匹配成功 → 查询用户信息 → 设置 session → 返回
```

### 5.3 会话管理流程

```
【获取会话列表】
GET /api/v1/chat/sessions/

       ← 200 OK
       {
         "code": 200,
         "msg": "success",
         "data": {
           "sessions": [
             {
               "session_id": 1,
               "title": "Python后端实习生",
               "last_message_time": "2026-04-28T10:30:00"
             }
           ]
         }
       }

【新建会话】
POST /api/v1/chat/sessions/
Body: { "title": "新会话" }

       ← 200 OK
       { "code": 200, "data": { "session_id": 2, "title": "新会话" } }

【获取会话消息】
GET /api/v1/chat/sessions/{session_id}/messages/

       ← 200 OK
       {
         "code": 200,
         "data": {
           "messages": [
             {
               "message_id": 1,
               "role": "user",
               "content": "我是计算机专业，想投贵阳的Python后端实习生",
               "thinking_process": null,
               "echarts_config": null,
               "created_at": "2026-04-28T10:30:00"
             },
             {
               "message_id": 2,
               "role": "assistant",
               "content": "# 贵阳Python后端实习生分析报告...",
               "thinking_process": "[工具调用] get-job-list({...})\n[工具返回] 20条JD数据...\n",
               "echarts_config": { "title": {...}, "series": [...] },
               "created_at": "2026-04-28T10:30:05"
```

### 5.4 Agent SSE 流式对话（核心业务流程）

这是整个系统最核心的数据流，涉及用户输入、Agent 思考、工具调用、数据解析、ECharts 生成、学习路径输出。

#### 5.4.1 请求格式

```
GET /api/v1/chat/agent/?session_id=1&content=我是计算机专业，想投贵阳的Python后端实习生&deep_thinking=true

请求头:
  Cookie: sessionid=xxx                    # 身份认证
  Accept: text/event-stream                # SSE 声明

无请求体（GET 请求，参数在 URL 中）
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| session_id | int | 是 | 会话ID |
| content | string | 是 | 用户输入的自然语言查询 |
| deep_thinking | bool | 否 | 是否开启深度思考模式，默认 `false` |

#### 5.4.2 响应格式（SSE 事件流）

```
Content-Type: text/event-stream
Cache-Control: no-cache
X-Accel-Buffering: no

响应体: SSE 事件流，按顺序发送以下事件:

event: content
data: #

event: content
data: 贵

event: content
data: 阳

event: content
data: ...

（中间穿插思考事件，仅 deep_thinking=true 时）
event: thinking
data: [工具调用] get-job-list({"keyword": "Python后端实习生"})

event: thinking
data: [工具返回] get-job-list: [{"name": "Python后端开发", "entityName": "...", ...}]

event: echarts
data: {"title": {"text": "技术栈出现频率Top15"}, "series": [...]}

event: end

event: error
data: 服务器内部错误，请稍后再试
```

**SSE 事件类型**:

| event | 说明 | data 内容 |
|-------|------|-----------|
| `content` | 最终回答的文字片段（流式） | 一个或多个字符 |
| `thinking` | 思考过程片段（仅开启深度思考时） | `[工具调用]` / `[工具返回]` / `[数据分析]` 格式的文本 |
| `echarts` | ECharts 图表配置 | JSON 格式的 ECharts 配置对象 |
| `end` | 流式输出结束标记 | 空 |
| `error` | 错误事件 | 错误描述文本 |

#### 5.4.3 内部处理流程

```
1. ChatAgentView.get() 接收请求
   ├── 验证用户登录状态（get_login_user_id）
   ├── 解析参数: session_id, content, deep_thinking
   └── 返回 StreamingHttpResponse（SSE 流）

2. stream_agent_chat_sse() 异步流处理
   │
   ├── [1] 验证会话归属权
   │
   ├── [2] 保存用户消息到数据库
   │   ChatMessage.objects.acreate(session_id, role="user", content)
   │
   ├── [3] 初始化 LLM（单例加载通义千问）
   │   llm = load_llm()
   │   → LLMService 加载 .env 中的 QWEN_API_KEY
   │   → 创建 ChatOpenAI(model='qwen3-max-2026-01-23')
   │
   ├── [4] 连接 MCP 客户端，获取招聘工具
   │   async with mcp_client:
   │       mcp_tools = mcp_client.get_tools()  # 获取 MCP-Jobs 的工具列表
   │       all_tools = list(mcp_tools) + [get_database]
   │
   ├── [5] 创建 LangGraph Agent
   │   agent = create_agent(model=llm, tools=all_tools, system_prompt=SYSTEM_PROMPT)
   │
   ├── [6] Agent 流式执行
   │   msg = {"messages": [{"role": "user", "content": content}]}
   │   async for event in agent.astream_events(msg, version='v2'):
   │
   │   ┌─ 事件分发 ──────────────────────────────────────────────────────┐
   │   │                                                                │
   │   │ on_tool_start (深度思考开启时):                                  │
   │   │   → 提取工具名和参数                                            │
   │   │   → 生成 thinking: "[工具调用] tool_name({...})"                │
   │   │   → SSE 发送 event: thinking                                   │
   │   │   → 累积到 full_thinking                                       │
   │   │                                                                │
   │   │ on_tool_end (深度思考开启时):                                    │
   │   │   → 提取工具返回结果（截取前200字符）                            │
   │   │   → 生成 thinking: "[工具返回] tool_name: ..."                  │
   │   │   → SSE 发送 event: thinking                                   │
   │   │   → 累积到 full_thinking                                       │
   │   │                                                                │
   │   │ on_chain_stream (深度思考开启时):                                │
   │   │   → 提取链式调用内容                                            │
   │   │   → 生成 thinking: "[数据分析] ..."                            │
   │   │   → SSE 发送 event: thinking                                   │
   │   │                                                                │
   │   │ on_chat_model_stream:                                          │
   │   │   → 提取 LLM 流式输出的文字片段                                │
   │   │   → SSE 发送 event: content                                    │
   │   │   → 累积到 full_content                                        │
   │   │   → 如果深度思考开启，也累积到 full_thinking                    │
   │   │                                                                │
   │   └─────────────────────────────────────────────────────────────────┘
   │
   ├── [7] 提取 ECharts 配置
   │   echarts_config = _extract_echarts_config(full_content)
   │   → 正则匹配 ```json { ... } ``` 代码块
   │   → 验证是否包含 series/xAxis 等 ECharts 关键属性
   │   → SSE 发送 event: echarts
   │
   ├── [8] 发送结束事件
   │   SSE 发送 event: end
   │
   └── [9] 保存 AI 回复到数据库
       ChatMessage.objects.acreate(
         session_id=session_id,
         role="assistant",
         content=full_content,
         thinking_process=full_thinking if deep_thinking else None,
         echarts_config=echarts_config,
       )
       ChatSession.objects.filter(session_id).aupdate(last_message_time=now)
```

## 六、统一响应格式

所有 API 响应均使用 `core/utils/response.py` 中的 `api_response()` 函数：

```python
def api_response(code=200, msg="success", data=None):
    return JsonResponse({
        "code": code,      # 业务状态码
        "msg": msg,        # 提示信息
        "data": data       # 数据内容（可为 None）
```

## 十一、前端数据接收与渲染

### 11.1 SSE 事件接收

```javascript
const eventSource = new EventSource(
  `/api/v1/chat/agent/?session_id=${sessionId}&content=${encodeURIComponent(query)}&deep_thinking=${deepThinking}`
);

// 接收最终回答文字
eventSource.addEventListener('content', (e) => {
  const chunk = e.data;
  // 追加到 Markdown 渲染器
  fullContent += chunk;
  renderMarkdown(fullContent);
});

// 接收思考过程（仅深度思考开启时）
eventSource.addEventListener('thinking', (e) => {
  const chunk = e.data;
  fullThinking += chunk;
  renderThinking(fullThinking);
});

// 接收 ECharts 配置
eventSource.addEventListener('echarts', (e) => {
  const config = JSON.parse(e.data);
  renderEChartsChart(config);
});

// 流结束
eventSource.addEventListener('end', () => {
  eventSource.close();
  // 保存消息到数据库
});

// 错误处理
eventSource.addEventListener('error', (e) => {
  eventSource.close();
  showError(e.data);
});
```

```
  │   ├── [工具调用] → 蓝色文字
  │   ├── [工具返回] → 绿色文字
  │   ├── [数据分析] → 橙色文字

  │
  ├── end 事件 → 完成
  │
  └── error 事件 → 错误提示
```

## 十三、安全注意事项

1. **密码明文存储**（待优化为 bcrypt 加密）
2. **CSRF 全豁免**（聊天、MCP、用户模块均使用 `@csrf_exempt`）

6. **API Key 存储在 .env 中**（正确做法）
