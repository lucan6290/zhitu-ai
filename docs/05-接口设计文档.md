### 2.2 时间格式统一说明

**数据库存储格式**：
- 类型：DATETIME
- 格式：`YYYY-MM-DD HH:MM:SS`
- 时区：服务器本地时区（东八区 UTC+8）

**接口返回格式**：
- 类型：String
- 格式：ISO 8601简化格式 `YYYY-MM-DDTHH:mm:ss`
- 示例：`"2026-04-28T10:30:00"`

**统一格式优势**：
- ✅ 后端转换极简：只需将空格替换为T
- ✅ 前端处理简单：JavaScript自动识别ISO格式
- ✅ 减少时区转换错误：在东八区场景下，省略时区信息不影响使用

**后端转换实现（Django）**：
```python
def format_datetime(dt):
    """
    将Django的datetime对象转换为ISO 8601简化格式
    
    Args:
        dt: Django的datetime对象
    
    Returns:
        str: ISO 8601简化格式的时间字符串
    
    Example:
        >>> from datetime import datetime
        >>> dt = datetime(2026, 4, 28, 10, 30, 0)
        >>> format_datetime(dt)
        '2026-04-28T10:30:00'
    """
    # 格式化为 YYYY-MM-DD HH:MM:SS
    dt_str = dt.strftime('%Y-%m-%d %H:%M:%S')
    # 将空格替换为T，符合ISO 8601格式
    return dt_str.replace(' ', 'T')

# 使用示例
user_info = {
    'user_id': 1,
    'user_name': '张三',
    'created_at': format_datetime(user.created_at)  # "2026-04-28T10:30:00"
}
```

**前端处理**：
```javascript
// JavaScript自动解析ISO 8601格式（即使省略时区）
const createdAt = new Date("2026-04-28T10:30:00");

// 格式化显示
const formattedTime = createdAt.toLocaleString('zh-CN', {
  year: 'numeric',
  month: '2-digit',
  day: '2-digit',
  hour: '2-digit',
  minute: '2-digit',
  second: '2-digit'
});
// 输出: "2026/04/28 10:30:00"

// 注意：在东八区环境下，JavaScript会自动按本地时区处理
```

### 3.4 聊天交互接口
#### 3.4.1 发送消息（SSE流式响应）
**接口地址**：`GET /api/v1/chat/agent/`

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `session_id` | BigInt | 是 | 会话ID（对应数据库chat_session.session_id） | 1 |
| `content` | String | 是 | 用户输入的消息内容（对应数据库chat_message.content） | "我是计算机专业，想投贵阳的Python后端实习生" |
| `deep_thinking` | Boolean | 否 | 是否开启深度思考模式，默认false | true |

**响应格式**：SSE（Server-Sent Events）流式响应

**事件类型说明**：
| 事件类型 | 触发时机 | 数据格式 | 说明 |
|----------|----------|----------|------|
| `thinking` | 大模型思考过程中（仅开启深度思考时） | 文本字符串 | 逐字流式输出思考内容 |
| `thinking_end` | 思考过程结束（仅开启深度思考时） | 空 | 表示思考完成，开始输出最终回答 |
| `content` | 输出最终回答内容时 | 文本字符串 | 逐字流式输出最终回答 |
| `echarts` | 生成ECharts配置时 | JSON对象 | 完整的ECharts 6.0配置 |
| `end` | 整个响应结束时 | 空 | 表示所有内容输出完成 |
| `error` | 发生错误时 | 错误信息字符串 | 输出错误描述，**之后不再发送任何事件** |

**SSE错误处理机制**：

**错误事件触发条件**：
1. MCP-Jobs服务不可用或超时
2. 大模型API调用失败
3. 数据解析异常
4. 网络连接中断
5. 服务器内部错误

**错误处理流程**：
```
正常流程：
thinking → thinking_end → content → echarts → end

错误流程（任何时候发生错误）：
... → error → [连接关闭]

示例：
event: thinking
data: 我需要调用search_jobs工具...

event: error
data: MCP服务暂时不可用，请稍后再试

[不再发送end事件，连接关闭]
```

**前端错误处理逻辑**：
```javascript
const eventSource = new EventSource('/api/v1/chat/agent/?session_id=1&content=...');

let hasError = false;

eventSource.addEventListener('error', (e) => {
    hasError = true;
    
    // 1. 停止显示加载状态
    hideLoading();
    
    // 2. 显示错误提示
    showErrorToast(e.data); // "MCP服务暂时不可用，请稍后再试"
    
    // 3. 保留用户输入内容（不清空输入框）
    // userInput.value 保持不变
    
    // 4. 允许用户重新发送
    enableSendButton();
    
    // 5. 关闭SSE连接
    eventSource.close();
});

eventSource.addEventListener('end', () => {
    if (!hasError) {
        // 只有在没有错误的情况下才处理正常结束
        hideLoading();
        saveMessageToDatabase();
    }
    eventSource.close();
});

// 注意：收到error事件后，不会再收到end事件
```

**后端错误处理逻辑**：
```python
async def chat_agent_stream(request):
    try:

            yield f"event: {event.type}\ndata: {event.data}\n\n"
```

### 3.5 MCP数据接口（内部使用）
#### 3.5.1 获取职位数据
**接口地址**：`GET /api/v1/mcp/jobs/`

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `keyword` | String | 是 | 岗位关键词（对应数据库mcp_jobs_cache.keyword） | "Python后端实习生" |
| `city` | String | 是 | 城市（对应数据库mcp_jobs_cache.city） | "贵阳" |
| `limit` | Int | 否 | 获取数量（对应数据库mcp_jobs_cache业务逻辑，默认20） | 20 |

**响应参数**：
| 参数名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `jobs` | Array | 结构化职位数据列表（对应数据库mcp_jobs_cache.jobs_data） | - |
| `from_cache` | Boolean | 是否来自缓存（业务逻辑字段，不存储在数据库中） | true |

**from_cache字段说明**：

**字段来源**：后端业务逻辑判断，不存储在数据库中，仅在接口响应时动态计算。

**判断逻辑**：
```
数据获取优先级：
1. 内存缓存（Redis） - 有效期1小时
2. MySQL缓存（mcp_jobs_cache表） - 有效期7天
3. MCP-Jobs API实时调用

from_cache返回规则：
- 来自内存缓存或MySQL缓存：true
- 来自MCP-Jobs API实时调用：false
```

**后端实现示例**：
```python
async def get_jobs(keyword, city, limit=20):
    """
    获取职位数据（带缓存）
    
    Returns:
        tuple: (jobs_data, from_cache)
    """
    cache_key = f"jobs:{keyword}:{city}"
    
    # 1. 检查内存缓存（有效期1小时）
    cached = redis.get(cache_key)
    if cached:
        return json.loads(cached), True
    
    # 2. 检查MySQL缓存（有效期7天）
    db_cache = MCPJobsCache.objects.filter(
        keyword=keyword,
        city=city,
        expire_at__gt=timezone.now()
    ).first()
    
    if db_cache:
        # 更新内存缓存
        redis.setex(cache_key, 3600, db_cache.jobs_data)
        return json.loads(db_cache.jobs_data), True
    
    # 3. 调用MCP-Jobs API获取实时数据
    jobs = await mcp_jobs_search(keyword, city, limit)
    
    # 4. 存储到MySQL和内存缓存
    MCPJobsCache.objects.create(
        keyword=keyword,
        city=city,
        jobs_data=json.dumps(jobs),
        expire_at=timezone.now() + timedelta(days=7)
    )
    redis.setex(cache_key, 3600, json.dumps(jobs))
    
    return jobs, False
```

## 四、错误码详细说明
| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数的类型、格式和必填项 |
| 401 | 未登录或登录已过期 | 清除本地缓存，重新登录 |
| 403 | 无权限访问 | 确认当前用户是否有访问该资源的权限 |
| 404 | 资源不存在 | 检查请求的URL和资源ID是否正确 |
| 429 | 请求过于频繁 | 降低请求频率，稍后再试 |
| 1001 | 邀请码无效 | 联系管理员获取有效的邀请码 |
| 1002 | 用户名已存在 | 更换其他用户名进行注册 |
| 1003 | 密码格式错误 | 密码长度需为6-20字符，且同时包含字母和数字 |
| 1004 | 登录账号或密码错误 | 检查登录账号（user_name/user_phone）和密码是否输入正确 |
| 1005 | 未识别到已注册人脸 | 请使用账号密码方式登录 |
| 2001 | 会话不存在 | 检查会话ID是否正确，或该会话已被删除 |
| 2002 | 无权访问该会话 | 只能访问自己创建的会话 |
| 3001 | MCP服务暂时不可用 | 稍后再试，系统会自动使用缓存数据 |
| 3002 | 未找到匹配的职位数据 | 尝试更换城市或岗位关键词 |
| 500 | 服务器内部错误 | 联系管理员排查问题 |
