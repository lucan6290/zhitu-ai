## 二、产品概述
### 2.1 产品定位
**职途AI**是一个**完全开源免费、面向计算机专业学生**的私有化岗位分析工具。基于实时招聘数据和大模型技术，自动解析多平台最新JD，生成精准的技术栈优先级和个性化学习路径。采用**邀请制注册**，保障服务稳定性，解决求职者"不知道学什么、先学什么"的核心痛点。

### 2.2 目标用户
**精准用户画像**：
- 年龄：20-24岁
- 身份：计算机科学与技术、软件工程、人工智能、大数据等相关专业大三、大四本科生，研二研究生
- 核心场景：准备暑期实习、秋招提前批、春招补录
- 核心诉求：快速了解目标岗位的真实技术要求，制定高效的学习计划，保存个人分析历史
- 技术爱好者、开发者，希望了解大模型的思考过程和工具调用逻辑

### 2.3 核心价值
1. **实时性**：基于MCP-Jobs获取最新30天内的招聘数据，拒绝过时的学习路线
2. **精准性**：针对具体城市和岗位进行分析，解决通用路线不贴合地域的问题
3. **高效性**：1分钟完成人工需要1.5小时的JD整理和分析工作
4. **可执行性**：输出分阶段学习计划，明确每个阶段的学习重点和验收标准
5. **可视化**：通过ECharts图表直观展示技术栈频率、分布等数据
6. **私有化**：用户数据本地存储，支持历史记录查询和多会话管理
7. **透明性**：新增深度思考功能，完整展示大模型的思考过程、工具调用和数据处理逻辑


## 四、功能需求
### 4.1 模块划分
v1.3.1版本包含以下7个核心模块：
1. **用户模块**
2. **聊天交互模块**
3. **深度思考展示模块**
4. **JD数据获取模块**
5. **智能分析模块**
6. **可视化展示模块**
7. **学习路径生成模块**

### 4.2 用户模块
#### 4.2.1 功能描述
提供完整的用户注册、登录、个人信息管理功能，采用**强制邀请码注册**机制，仅允许授权用户使用系统。支持账号密码登录和人脸识别登录两种方式。

#### 4.2.2 子功能列表
| 子功能 | 功能描述 |
|--------|----------|
| 用户注册 | 填写姓名、密码、邀请码，可选采集人脸，邀请码验证通过后方可注册 |
| 账号密码登录 | 使用用户名+密码登录 |
| 人脸识别登录 | 使用摄像头抓拍人脸，与已注册人脸比对登录 |
| 个人中心 | 查看/编辑用户信息（姓名、年龄、电话、密码） |
| 退出登录 | 清除登录状态，跳转至登录页 |

#### 4.2.3 输入输出
| 功能 | 输入 | 输出 |
|------|------|------|
| 用户注册 | 姓名（6-20字符）、密码（6-20字符，含字母+数字）、邀请码（必填）、人脸图片（可选，base64） | 注册成功/失败提示，跳转至登录页 |
| 账号密码登录 | 用户名/手机号、密码 | 登录成功/失败提示，跳转至首页 |
| 人脸识别登录 | 人脸图片（base64） | 登录成功/失败提示，跳转至首页 |
| 个人信息更新 | 年龄、电话、新密码 | 更新成功/失败提示 |

#### 4.2.4 业务规则
1. **强制邀请码**：注册必须填写有效邀请码，无邀请码无法注册
2. **用户名唯一**：系统中用户名不可重复
3. **人脸可选**：注册时可选择是否采集人脸，未采集人脸的用户只能使用账号密码登录
4. **密码规则**：密码长度6-20字符，必须同时包含字母和数字
5. **登录状态**：登录状态有效期7天，7天后自动退出
6. **权限控制**：未登录用户无法访问任何功能，所有接口必须携带用户身份信息

#### 4.2.5 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 邀请码无效 | 提示"邀请码无效，请联系管理员获取" |
| 用户名已存在 | 提示"用户名已被注册，请更换" |
| 密码格式错误 | 提示"密码长度6-20字符，必须同时包含字母和数字" |
| 人脸比对失败 | 提示"未识别到已注册人脸，请使用账号密码登录" |
| 登录状态过期 | 自动跳转至登录页，提示"登录已过期，请重新登录" |

### 4.3 聊天交互模块
#### 4.3.1 功能描述
提供自然语言聊天界面，支持多会话管理、上下文对话、历史记录查询，新增**深度思考模式**，可选择展示大模型的完整思考过程。所有对话数据、思考过程永久存储在MySQL数据库中。

#### 4.3.2 子功能列表
| 子功能 | 功能描述 |
|--------|----------|
| 新建会话 | 创建新的聊天会话，自动生成会话标题 |
| 切换会话 | 点击左侧历史会话列表，切换到对应会话 |
| 删除会话 | 删除指定会话及其所有消息 |
| 上下文对话 | 保留当前会话的所有历史消息，支持多轮连续对话 |
| 历史记录加载 | 登录后自动加载用户所有历史会话 |
| 流式输出 | AI回复文字内容逐字流式输出 |
| 深度思考开关 | 全局开关，控制是否展示大模型思考过程，默认关闭 |
| 思考过程流式输出 | 开启深度思考后，逐字流式输出大模型的思考内容 |
| 思考框折叠/展开 | 思考完成后自动折叠，支持手动展开/折叠查看 |
| 思考过程历史保存 | 所有思考过程自动保存到数据库，历史会话中可查看 |

#### 4.3.3 输入输出
| 功能 | 输入 | 输出 |
|------|------|------|
| 发送消息（深度思考开启） | 用户自然语言查询 | 1. 流式输出思考过程<br>2. 思考完成后自动折叠思考框<br>3. 流式输出最终文字分析结论<br>4. 渲染ECharts图表<br>5. 输出学习路径 |
| 发送消息（深度思考关闭） | 用户自然语言查询 | 1. 直接流式输出最终文字分析结论<br>2. 渲染ECharts图表<br>3. 输出学习路径 |
| 切换深度思考开关 | 开关状态变更 | 1. 开启：后续所有对话都展示思考过程<br>2. 关闭：后续所有对话都不展示思考过程<br>3. 不影响已发送的历史消息 |
| 新建会话 | 无 | 创建新会话，清空聊天区域 |
| 切换会话 | 会话ID | 加载该会话的所有历史消息 |
| 删除会话 | 会话ID | 删除该会话及其所有消息，刷新会话列表 |

#### 4.3.4 业务规则
1. **多会话隔离**：不同会话之间上下文完全隔离，互不影响
2. **上下文保留**：每个会话保留所有历史消息，大模型可访问完整上下文
3. **自动保存**：用户发送的每条消息和AI的每条回复都自动保存到MySQL
4. **会话标题**：新会话的第一条消息自动作为会话标题，支持手动修改（后续迭代）
5. **会话排序**：历史会话列表按最后更新时间降序排列
6. **数据归属**：每个用户只能访问自己的会话和消息数据
7. **全局开关**：深度思考开关是全局设置，对所有会话生效，状态保存在用户本地localStorage
8. **流式顺序**：开启深度思考时，严格按照"思考过程 → 最终回答"的顺序流式输出，思考过程未完成时不输出最终回答
9. **自动折叠**：思考过程输出完成后，思考框自动折叠，显示"已完成思考（点击展开查看）"
10. **手动控制**：用户可随时点击思考框的折叠/展开按钮，查看或隐藏思考过程
11. **历史保留**：所有历史会话的思考过程都永久保存，切换到历史会话时，思考框保持当时的折叠/展开状态
12. **工具调用展示**：思考过程中需明确展示大模型调用了哪些工具、传入了什么参数、返回了什么结果

#### 4.3.5 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 会话不存在 | 提示"会话不存在"，自动跳转到最新会话 |
| 历史记录加载失败 | 提示"加载历史记录失败，请刷新页面重试" |
| 消息发送失败 | 提示"消息发送失败，请重试"，保留输入内容 |
| 思考过程输出中断 | 显示"思考过程中断，点击重试"，重试后重新输出完整思考过程 |
| 思考过程格式错误 | 过滤无效内容，继续输出剩余部分，不影响最终回答 |
| 历史思考过程加载失败 | 显示"无法加载思考过程"，但仍展示最终回答 |

### 4.4 深度思考展示模块
#### 4.4.1 功能描述
专门用于展示大模型的思考过程，包括问题理解、工具调用决策、数据处理逻辑、分析过程和最终结论生成步骤。

#### 4.4.2 输入输出
| 输入 | 输出 |
|------|------|
| LangGraph Agent返回的流式思考事件 | 格式化后的思考过程文本，包含：<br>- 思考步骤编号<br>- 工具调用信息（工具名、参数、返回结果）<br>- 数据分析过程<br>- 结论生成逻辑 |

#### 4.4.3 业务规则
1. **格式规范**：思考过程采用统一的格式化输出，不同类型的内容使用不同的样式：
   - 普通思考：灰色文字
   - 工具调用：蓝色文字，前缀`[工具调用]`
   - 工具返回：绿色文字，前缀`[工具返回]`
   - 数据分析：橙色文字，前缀`[数据分析]`
2. **流式输出**：思考过程逐字流式输出，与大模型的思考速度保持一致
3. **折叠状态**：思考框默认展开，思考完成后自动折叠，折叠后只显示标题和展开按钮
4. **高度限制**：思考框最大高度为300px，超过时显示滚动条
5. **复制功能**：支持一键复制完整的思考过程文本

#### 4.4.4 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 收到无效的思考事件 | 忽略该事件，继续处理后续事件 |
| 思考过程过长（>5000字符） | 自动截断，显示"思考过程过长，点击查看完整内容"，点击后展开全部 |

### 4.5 JD数据获取模块
#### 4.5.1 功能描述
调用MCP-Jobs服务，从BOSS直聘、猎聘、智联招聘三个平台获取最新的结构化JD数据，新增MySQL两级缓存机制，减少第三方API调用，提高响应速度。

#### 4.5.2 输入输出
| 输入 | 输出 |
|------|------|
| 岗位关键词（如"Python后端实习生"） | 结构化JD列表，包含：<br>- job_id、title、company、city<br>- salary_min、salary_max<br>- description、requirements |
| 城市（如"贵阳"） | |
| 数量（默认20条） | |

#### 4.5.3 业务规则
1. 默认获取最近30天内发布的岗位
2. 优先获取薪资明确、描述完整的岗位
3. 自动去重，按`title+company`过滤重复岗位
4. **两级缓存**：
   - 一级缓存：内存缓存，有效期1小时
   - 二级缓存：MySQL缓存，有效期7天
5. **缓存命中逻辑**：相同关键词+城市的查询，优先返回缓存数据
6. **缓存更新**：缓存过期后自动重新获取最新数据并更新缓存
7. **数据存储**：所有从MCP-Jobs获取的原始JD数据都永久存储在MySQL中，用于后续离线分析
8. 最多同时从3个平台获取数据，禁用前程无忧（数据质量较低）

#### 4.5.4 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| MCP-Jobs API密钥无效 | 记录错误日志，返回服务不可用提示 |
| 平台限流 | 自动重试3次，每次间隔2秒，仍失败则返回提示 |
| 返回数据为空 | 提示用户更换城市或岗位关键词 |
| 数据格式错误 | 过滤无效数据，使用剩余有效数据进行分析 |
| MCP-Jobs服务不可用 | 检查MySQL缓存，如有缓存数据则返回缓存数据并提示"数据为7天内缓存" |
| 缓存数据过期且服务不可用 | 提示"暂时无法获取最新数据，请稍后再试" |

### 4.6 智能分析模块
#### 4.6.1 功能描述
使用通义千问qwen3-max批量解析JD数据，提取技术栈、岗位职责、软技能等信息，进行统计分析和优先级分级。

#### 4.6.2 输入输出
| 输入 | 输出 |
|------|------|
| 结构化JD列表 | 分析结果JSON，包含：<br>- 技术栈分级（必备/重要/加分）<br>- 经验要求分布<br>- 学历要求分布<br>- 薪资范围分布<br>- 高频岗位职责<br>- 高频软技能 |

#### 4.6.3 业务规则
1. 技术栈分级标准：
   - 必备：出现频率 > 70%
   - 重要：30% ≤ 频率 ≤ 70%
   - 加分：频率 < 30%
2. 技术栈按领域分类：编程语言、后端框架、数据库、缓存、工具、云服务、其他
3. 统计结果保留整数，频率计算精确到个位
4. 自动过滤非技术词汇（如"计算机相关专业"、"本科及以上"）
5. 统一技术别名（如"Django REST framework"合并到"Django"）

#### 4.6.4 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 大模型调用失败 | 自动重试2次，仍失败则返回提示 |
| 返回JSON格式错误 | 使用备用解析逻辑，提取关键信息 |
| 有效JD数量不足5条 | 提示："有效岗位数据较少，分析结果仅供参考" |

### 4.7 可视化展示模块
#### 4.7.1 功能描述
根据分析结果生成标准ECharts 6.0配置，前端渲染图表，直观展示数据分析结果。

#### 4.7.2 输入输出
| 输入 | 输出 |
|------|------|
| 智能分析结果JSON | 3个ECharts配置：<br>1. 柱状图：技术栈出现频率Top15<br>2. 饼图：技术栈领域分布<br>3. 柱状图：经验要求分布 |

#### 4.7.3 业务规则
1. 图表必须符合ECharts 6.0官方规范，无废弃属性
2. 图表标题清晰，坐标轴有明确的名称和单位
3. 使用ECharts默认配色，不自定义样式
4. 图表支持响应式，随窗口大小自动调整
5. 鼠标悬停显示详细数据

#### 4.7.4 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| ECharts配置生成失败 | 跳过该图表，继续展示其他内容 |
| 图表渲染失败 | 显示文字版数据，不影响整体流程 |

### 4.8 学习路径生成模块
#### 4.8.1 功能描述
结合用户的专业背景和分析结果，生成分阶段的个性化学习路径规划。

#### 4.8.2 输入输出
| 输入 | 输出 |
|------|------|
| 用户专业背景（如"计算机专业"） | 分阶段学习计划：<br>- 第1周：基础必备技能<br>- 第2周：核心技能深化<br>- 第1个月：项目实战 |
| 智能分析结果JSON | |

#### 4.8.3 业务规则
1. 学习计划分为3个阶段，总时长1个月
2. 每个阶段明确学习重点、推荐资源和验收标准
3. 优先安排必备技术栈，其次是重要技术栈
4. 结合用户专业背景调整难度，计算机专业可适当加快进度
5. 推荐的项目实战要贴合目标岗位的实际需求

#### 4.8.4 异常处理
| 异常场景 | 处理方式 |
|----------|----------|
| 无法识别用户专业 | 使用通用学习计划模板 |
| 技术栈数量过多 | 优先推荐最重要的10个技术点 |

### 4.9 数据存储模块
#### 4.9.1 功能描述
所有用户数据、会话数据、消息数据都永久存储在MySQL数据库中，确保数据持久化和可追溯。思考过程根据用户设置的深度思考开关决定是否存储。

#### 4.9.2 数据库表设计
| 表名 | 说明 | 核心字段 |
|------|------|----------|
| `user_info` | 用户信息表 | id, user_name, user_age, user_phone, user_pwd, created_at, updated_at |
| `chat_session` | 聊天会话表 | id, user_id, title, last_message_time, created_at, updated_at |
| `chat_message` | 聊天消息表 | id, session_id, user_id, role, content, thinking_process, echarts_config, created_at |
| `mcp_jobs_cache` | MCP数据缓存表 | id, keyword, city, jobs_data, created_at, expire_at |

**字段说明**：
- `thinking_process`：TEXT类型，存储该条AI消息对应的完整思考过程
  - 开启深度思考时：存储完整思考过程文本
  - 关闭深度思考时：字段值为NULL

#### 4.9.3 数据存储策略

**永久存储的数据**：
1. 用户基本信息（user_info表）
2. 所有聊天会话（chat_session表）
3. 所有用户消息和AI回复的最终内容（chat_message表的content字段）
4. 所有ECharts图表配置（chat_message表的echarts_config字段）
5. 所有MCP原始数据（mcp_jobs_cache表）

**条件存储的数据**：
1. 思考过程（chat_message表的thinking_process字段）：
   - **开启深度思考**：存储完整思考过程，包括问题理解、工具调用、数据分析等所有步骤
   - **关闭深度思考**：字段值为NULL，不占用存储空间

**存储规则说明**：
- 思考过程的存储与否由用户发送消息时的深度思考开关状态决定
- 开关状态保存在前端localStorage中，每次发送消息时读取当前状态
- 已存储的历史思考过程不会因开关状态变更而删除
- 删除会话或消息时，级联删除对应的思考过程

#### 4.9.4 业务规则
1. **外键约束**：所有关联表都建立外键约束，删除用户时级联删除其所有会话和消息
2. **级联删除**：删除会话或消息时，级联删除对应的思考过程和ECharts配置
3. **索引优化**：对常用查询字段建立索引（user_id, session_id, keyword+city）
4. **数据备份**：每日自动备份数据库，防止数据丢失
5. **隐私保护**：不存储任何用户敏感数据，密码采用bcrypt加密存储（后续优化）

## 五、非功能需求
### 5.1 性能要求
| 指标 | 要求 |
|------|------|
| 页面首次加载时间 | ≤ 2秒 |
| 登录/注册接口响应时间 | ≤ 1秒 |
| 历史记录加载时间 | ≤ 2秒（最多加载50条会话，包含思考过程） |
| 端到端完整流程响应时间 | ≤ 35秒（开启深度思考）<br>≤ 30秒（关闭深度思考）<br>≤ 5秒（缓存命中） |
| 思考过程流式输出首字延迟 | ≤ 3秒 |
| SSE最终回答首字延迟 | ≤ 5秒（思考完成后） |
| 图表渲染时间 | ≤ 1秒/个 |
| 缓存命中率 | ≥ 85% |
| 单用户并发请求数 | ≤ 3 |
| 数据库查询响应时间 | ≤ 100ms |

### 5.2 安全要求
1. **用户认证**：所有接口必须进行用户身份验证，未登录用户无法访问任何功能
2. **权限控制**：用户只能访问自己的会话、消息和思考过程，禁止越权访问
3. **输入校验**：所有用户输入必须进行严格校验，防止XSS攻击和SQL注入
4. **API密钥保护**：所有第三方API密钥存储在环境变量中，不得硬编码
5. **数据加密**：用户密码采用bcrypt加密存储（后续优化），聊天数据和思考过程明文存储
6. **会话安全**：采用HttpOnly Cookie存储登录凭证，防止XSS窃取
7. **调用限制**：单用户每日最多调用20次MCP-Jobs服务，防止滥用

### 5.3 兼容性要求
| 类别 | 要求 |
|------|------|
| 浏览器兼容性 | 支持Chrome、Edge、Firefox最新两个稳定版本<br>不支持IE浏览器 |
| 设备兼容性 | 优先支持桌面端（1920×1080及以上分辨率）<br>基础支持平板端（768×1024及以上分辨率）<br>不支持手机端 |
| 操作系统兼容性 | 支持Windows 10+、macOS 12+、Ubuntu 20.04+ |
