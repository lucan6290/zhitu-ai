## 二、本地开发架构原则
### 2.1 核心原则
| 原则 | 说明 |
|------|------|
| **最小化依赖** | 所有服务本地运行，不依赖任何云服务，开箱即用 |
| **调试友好** | 保留完整的调试信息，关闭生产级安全限制，便于问题排查 |
| **与生产环境对齐** | 技术栈、代码结构、接口规范与生产环境完全一致，避免"本地能跑，线上崩" |
| **自动化** | 尽量使用自动化脚本，减少手动配置步骤 |
| **模块化** | 代码高度模块化，便于功能扩展和单元测试 |

### 2.2 本地开发环境要求

| **第三方服务层** | 外部API调用 | 通义千问API + 本地运行的mcp-jobs服务 |

### 5.3 本地开发环境搭建步骤
#### 1. 克隆项目
```bash
git clone <项目仓库地址>
cd agent_face_pro
```

#### 2. 后端环境搭建
```bash
# 创建虚拟环境
python -m venv venv
# 激活虚拟环境
# Windows: venv\Scripts\activate
# Linux/macOS: source venv/bin/activate

# 安装依赖
cd back-end
pip install -r requirements.txt

# 初始化数据库
python manage.py makemigrations
python manage.py migrate

# 创建管理员账号（可选，用于Django Admin）
python manage.py createsuperuser

# 启动后端开发服务器
python manage.py runserver 8888
```

#### 3. 前端环境搭建
```bash
cd front-end
# 安装依赖
npm install

# 启动前端开发服务器
npm run dev
```

#### 4. 本地服务启动
```bash
# 启动MySQL服务（本地安装）
# 启动Redis服务（本地安装）

# 启动MCP-Jobs服务
npx mcp-jobs@1.4.0
```

#### 5. 配置环境变量
在`back-end/config/settings/development.py`中配置以下环境变量：
```python
# 通义千问API配置
DASHSCOPE_API_KEY = "your-dashscope-api-key"

# MCP-Jobs配置
MCP_JOBS_API_KEY = "your-mcp-jobs-api-key"

# 数据库配置
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.mysql",
        "NAME": "agent_face_sql",
        "USER": "root",
        "PASSWORD": "your-mysql-password",
        "HOST": "localhost",
        "PORT": "3306",
        "CHARSET": "utf8mb4",
    }
}

# Redis配置
CACHES = {
    "default": {
        "BACKEND": "django_redis.cache.RedisCache",
        "LOCATION": "redis://localhost:6379/1",
        "OPTIONS": {
            "CLIENT_CLASS": "django_redis.client.DefaultClient",
        }
    }
}

# Session配置
SESSION_ENGINE = "django.contrib.sessions.backends.cache"
SESSION_CACHE_ALIAS = "default"

# 开发模式配置
DEBUG = True
ALLOWED_HOSTS = ["localhost", "127.0.0.1"]
```

## 六、本地开发安全配置
### 6.1 开发环境安全原则
| 原则 | 说明 |
|------|------|
| **关闭生产级安全限制** | 方便调试，如关闭CSRF保护、允许跨域 |
| **保留基础安全措施** | 输入校验、SQL注入防护等基础安全不能关 |
| **不提交敏感信息** | API密钥、数据库密码等敏感信息不提交到Git |
| **数据隔离** | 开发环境使用独立的数据库，不与生产环境混用 |

### 6.2 本地开发安全配置
```python
# back-end/config/settings/development.py
# 开发模式
DEBUG = True

# 允许所有主机访问
ALLOWED_HOSTS = ["*"]

# 关闭CSRF保护（开发环境）
MIDDLEWARE = [
    # 'django.middleware.csrf.CsrfViewMiddleware',
]

# 允许跨域（开发环境）
CORS_ALLOW_ALL_ORIGINS = True
CORS_ALLOW_CREDENTIALS = True

# 详细错误页面
DEBUG_PROPAGATE_EXCEPTIONS = True

# 日志级别设置为DEBUG
LOGGING = {
    "version": 1,
    "disable_existing_loggers": False,
    "handlers": {
        "console": {
            "class": "logging.StreamHandler",
        },
    },
    "root": {
        "handlers": ["console"],
        "level": "DEBUG",
    },
}
```

### 6.3 敏感信息管理
1. **使用环境变量**：所有敏感信息（API密钥、数据库密码）都通过环境变量配置，不硬编码在代码中
2. **.gitignore文件**：确保`.env`、`local_settings.py`等包含敏感信息的文件不被提交到Git
3. **本地配置文件**：创建`local_settings.py`文件，覆盖默认配置，该文件不提交到Git

## 七、扩展性设计（代码层面）
### 7.1 模块化设计原则
| 原则 | 说明 |
|------|------|
| **单一职责** | 每个模块只负责一个功能，如认证模块只处理注册登录 |
| **接口隔离** | 模块之间通过接口交互，不直接依赖内部实现 |
| **依赖注入** | 依赖通过参数传入，便于单元测试 |
| **配置驱动** | 可配置的内容都放在配置文件中，不硬编码 |

### 7.2 代码扩展性设计
#### 1. AI Agent扩展
- 所有工具都通过MCP架构集成，新增工具只需在MCP服务中添加函数，无需修改Agent核心代码
- 系统提示词单独放在`system_prompt.py`文件中，便于修改和扩展
- Agent加载逻辑封装在`loader.py`中，支持动态加载不同的Agent

#### 2. 前端组件扩展
- 公共组件单独放在`components`目录中，如`EChartsRenderer.vue`、`ThinkingBox.vue`
- 页面组件按功能划分，每个页面一个独立的Vue文件
- API请求统一封装在`api/index.js`中，便于统一管理

#### 3. 数据库扩展
- 所有模型都继承自Django的`models.Model`，便于扩展字段
- 外键约束明确，数据关系清晰
- 索引设计合理，支持未来数据量增长

### 7.3 未来功能扩展路径
| 功能 | 扩展方式 | 影响范围 |
|------|----------|----------|
| 新增MCP工具 | 在MCP服务中添加工具函数 | 无影响，Agent自动发现 |
| 新增图表类型 | 扩展`EChartsRenderer.vue`组件 | 仅影响前端 |
| 新增用户字段 | 在`UserInfo`模型中添加字段 | 影响用户模块和个人中心 |
| 支持更多大模型 | 在`loader.py`中添加大模型支持 | 影响AI Agent模块 |
| 新增第三方登录 | 在认证模块中添加登录接口 | 影响认证模块和登录页 |
